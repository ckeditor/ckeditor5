Added a new shape for `config.root.element` (and `config.roots.<rootName>.element` in the multi-root editor) that lets integrators describe the editable root element without supplying an existing DOM node. The configuration now accepts a tag-name string (for example `'h1'`) or a `ViewRootElementDefinition` object that specifies a tag name, CSS classes, inline styles, and arbitrary attributes. The `name` property is optional and defaults to `'div'`, so integrators can keep the default element and only add classes or attributes. Inputs are normalized to a canonical form: class names provided as whitespace-separated strings are split into individual tokens, and `attributes.class` is lifted into the `classes` array. The `<textarea>` and `<input>` tag names are rejected because they cannot host a rich-text editable.
