The `config.initialData` property is no longer modified by the editor during initialization. Previously, if a custom plugin changed `config.initialData` at startup, that change would affect the data loaded into the editor. If you relied on this behavior in a custom plugin, migrate to using `config.roots.main.initialData` (for a single-root editor) or `config.roots.<rootName>.initialData` (for a multi-root editor) to set the initial data for each root instead.
